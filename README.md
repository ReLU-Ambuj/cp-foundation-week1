# Competitive Programming Foundation: Week 1

## Purpose of Week 1
To build an unshakable foundation in Core Syntax, Variables, Input/Output, Arithmetic, and Type Casting. This week bridges the gap between basic syntax knowledge and competitive programming readiness.

## Learning Objectives
- Master Python variables from a memory perspective.
- Understand implicit vs. explicit type casting.
- Predict and manipulate operator precedence confidently.
- Read and decipher stack traces immediately.
- Transition from reading code to building logical muscle memory.

## How to use this repository
This repository is self-contained. You do not need external websites.
1. **Theory**: Study the deep dives in `notes/` with pen and paper.
2. **Exercises**: Implement the 20 micro-exercises in `day1/exercises/` manually.
3. **Debugging**: Intentionally execute and study the files in `day1/debugging/`.
4. **Platform Problems**: Attempt the 10 graded problems in `day1/platform_problems/`.
5. **Reflection**: Take the speed round without IDE assistance.

## Study Flow (Hour-by-Hour Plan)
- **Hour 0–1**: Setup Properly (Structure, Repo, Debugger)
- **Hour 1–3**: Theory Deep Dive (`notes/`) - Write notes manually.
- **Hour 3–6**: 20 Micro Exercises (`day1/exercises/`) - Implement logic. Commit after.
- **Hour 6–8**: Debugging Practice (`day1/debugging/`) - Read stack traces.
- **Hour 8–12**: 10 Easy Problems (`day1/platform_problems/`) - Max 20m per problem.
- **Hour 12–14**: Reflection + Speed Round (`reflection/speed_round.md`) - Reinforce memory.

## Commit Strategy
Commit frequently to build the habit. 
- Use imperative tone: `feat: implement simple interest calculator`
- Group logical changes: `docs: add deep dive notes on operator precedence`

## Rules for the Speed Round
1. No looking back at previous code.
2. No IDE autocomplete, Copilot, or IntelliSense.
3. Pass all self-evaluations truthfully.
